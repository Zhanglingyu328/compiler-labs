服务器运行命令

cd /home/dfa_lab/zly/lab5 make clean make make summary # 生成所有 AST 图片 for f in output/*.ast.dot; do dot -Tpng "$f" -o "${f%.dot}.png" done # 重新生成汇总表，让 png 列显示 yes ./scripts/summary.sh # 启动网页服务 python3 -m http.server 8000 # 浏览器打开 http://服务器IP:8000/lab5.html

页面使用说明

  1. 先在服务器运行 make summary，生成 output/summary.csv 和 output/*.ir.txt。
  2. 运行 dot -Tpng 命令生成 output/*.ast.png。
  3. 启动 python3 -m http.server 8000。
  4. 进入本页面后点击“自动读取 summary.csv”。
  5. 进入“结果查看器”，选择某个样例，分别查看 AST、符号表、错误报告、IR 和 AST 图片。
如果学校服务器端口无法从外网访问，可以用浏览器直接打开 HTML，然后手动上传 summary.csv、AST、IR、PNG 文件进行展示。

必做部分

在 SLR(1) 语法分析基础上加入语义分析框架。程序在分析过程中构建 AST，维护符号表，并输出语义错误报告。支持变量声明、赋值语句、return 语句、函数声明、函数调用等结构，能够检测未声明变量、未声明函数、重复声明和类型不匹配等语义错误。

选做一：AST 可视化

在 AST 文本输出的基础上，额外输出 Graphviz DOT 文件，并通过 dot 工具转换为 PNG 图片，便于直观展示程序的语法结构。

选做二：批量测试汇总

编写 summary.sh，对多个 .src 示例程序批量运行，并自动统计 AST、DOT、PNG、IR、符号表、错误数量和警告数量。

选做三：四元式中间代码

在 AST 基础上初步生成四元式 IR，例如变量声明、数组声明、赋值、函数调用、return、print、if、while 和表达式计算。该部分可作为实验六中间代码生成的基础。
