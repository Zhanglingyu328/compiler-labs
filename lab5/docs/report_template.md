# 实验五报告模板

## 一、实验目的

在 SLR(1) 语法分析基础上增加语义动作，通过规约动作构造 AST，维护符号表，并完成基本语义检查。

## 二、输入输出

输入：

- 实验四生成的 SLR(1) ACTION/GOTO 分析表
- 实验二输出的 Token 流
- 嵌入语义动作的产生式

输出：

- AST 文本树和 DOT 可视化文件
- 符号表
- 语义错误报告

## 三、核心数据结构

### ASTNode

字段包括：节点名称、节点值、类型、place 临时变量名、子节点列表。

### Attribute

属性栈元素，包含 AST 节点、类型、词素值、place。

### SymbolTable

采用作用域栈结构，每层作用域是一个映射表，记录变量、数组、函数和参数。

## 四、语义动作设计

- 类型产生式：传递类型属性。
- 变量声明：插入符号表，检查重复声明。
- 标识符引用：查询符号表，检查未声明错误。
- 表达式规约：生成 AST，并计算表达式类型。
- 赋值语句：检查左右类型是否兼容。
- return 语句：检查返回类型与函数声明类型是否一致。
- 函数调用：检查函数是否声明、参数个数和参数类型。

## 五、测试结果

建议展示：

- `examples/1.src`：正常变量声明、赋值、return。
- `examples/5.src`：未声明变量错误。
- `examples/6.src`：未声明函数错误。
- `examples/11.src`：函数参数个数/类型错误。
- `examples/12.src`：数组声明和数组元素赋值。

## 六、选做拓展

本实验额外实现：

1. AST DOT 输出。
2. AST 节点保留 place 字段，方便实验六生成中间代码。
3. 表达式节点生成临时变量名。
4. 逻辑运算符和一元负号作为扩展语法支持。
