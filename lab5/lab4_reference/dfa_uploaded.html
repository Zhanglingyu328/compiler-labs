实验说明与报告亮点

本平台将第一次实验的 DFA 自动机、第二次实验的类 C 语言词法分析器、第三次实验的 LR(0) 项目集规范族构造、第四次实验的 SLR(1) 分析表生成融合，展示从字符识别、Token 扫描、项目集构造到 ACTION/GOTO 表和输入串分析的完整过程。

DFA 五元组 字符串判定 合法串枚举 Token 扫描 错误定位 统计分析 LR(0) 项目集 Closure / Goto FIRST / FOLLOW SLR(1) ACTION/GOTO 输入串分析 acc Flex / Bison 选做 冲突检测
模块一：DFA 可视化实验
用于展示确定有限自动机的五元组、状态转移、字符串判定和合法字符串枚举。支持内置案例和自定义 DFA。
模块二：类 C 词法分析器
用于扫描类 C 语言源代码，识别关键字、标识符、整数、实数、运算符、界符，并输出 token 序列。
DFA 与 scanner 的关系
词法分析器本质上可以看作多个 DFA 的组合。每类 token 都可以由正则式描述，再进一步转换为 NFA/DFA 进行识别。
模块三：LR(0) 项目集规范族
用于读取表达式文法，自动增广文法，计算 Closure 闭包和 Goto 转移，生成 LR(0) 项目集规范族，并检测移进-归约冲突或归约-归约冲突。
高分拓展点
本系统加入了自定义 DFA、状态转移表、token 表格展示、行列号定位、词法错误恢复、token 数量统计、LR(0) 状态图、SLR(1) ACTION/GOTO 表、输入串分析过程、JSON 导出和 Web 可视化页面。
模块四：SLR(1) 分析表与输入串分析
用于在 LR(0) 项目集规范族基础上计算 FIRST/FOLLOW 集，生成 SLR(1) ACTION/GOTO 表，并展示 id + id * id 的移进、归约、acc 成功过程。
报告建议写法：本实验在命令行程序基础上进一步实现了 Web 综合可视化平台，使 DFA 理论实验、类 C 语言词法分析器、LR(0) 项目集规范族构造和 SLR(1) 语法分析表生成形成串联，体现了正则式、自动机、词法分析和语法分析之间的联系。
