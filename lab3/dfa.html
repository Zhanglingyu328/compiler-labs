实验三报告亮点写法

下面内容可以直接用于实验报告的“系统设计”“实验结果分析”“选做部分说明”。

模块化设计工程化
本实验采用文法输入、项目集构造、状态转移展示、冲突检测和 Token 串联展示等模块化设计，使语法分析实验不再停留在静态输出，而是形成可交互的编译器前端展示平台。
LR(0) 核心算法必做
程序自动读取 EBNF 风格文法，完成文法增广，并从初始项目 E' -> ·E 出发，递归计算 Closure 闭包，再通过 Goto 函数生成项目集规范族。
冲突检测分析
系统会检测每个状态中是否同时存在移进项目和归约项目，或是否存在多个归约项目，从而判断文法是否属于严格 LR(0) 文法，并为后续 SLR(1) 分析表生成提供依据。
实验串联选做
在选做部分中，Scanner 将源程序字符流转换为 token 序列，具体变量归约为 id，数字归约为 num，使实验二输出能够与实验三文法终结符集合对接。
建议截图顺序：页面首页 → 输入文法 → 生成项目集 I0 → Goto 表 → 状态图 → 冲突检测 JSON → Scanner 串联 token 输出。
